# Agent Flow

## MVP Flow

1. Orchestrator starts intake and invokes the Interview Skill.
2. Story Strategy Agent extracts story units and claim gaps.
3. Media Design Agent reviews assets and proposes design/motion options.
4. Site Planning Agent creates sitemap, sections, CTA model, and copy brief.
5. Build QA Agent generates the site from `website-template`, reviews it, and patches issues.

## Stage Outputs

- Intake produces canonical profile updates and open questions.
- Story produces narrative arc, story units, and claim records.
- Media/design produces asset reviews, theme options, and motion plan.
- Planning produces approved site plan and copy brief.
- Build produces changed files and a QA report.

## Approval Checkpoints

- Public-facing claims require approval.
- Personal stories require visibility status.
- Design direction should be selected before final copy.
- Final build should pass QA before deployment.
