# Generation Pipeline

The generation pipeline turns raw human input into a complete website solution.

## Stage 1: Intake

Goal: Understand the user, target audience, desired outcome, tone, constraints, and available material.

Primary skills:

- `skills/interview`
- `skills/claim-checking`

Outputs:

- Initial canonical profile.
- Target audience.
- Primary site goal.
- Open questions.
- Early claim records.

## Stage 2: Story Strategy

Goal: Convert work and life material into a narrative structure.

Primary skills:

- `skills/story-extraction`
- `skills/claim-checking`
- `skills/site-architecture`

Outputs:

- Core identity.
- Narrative arc.
- Signature themes.
- Story units.
- Case study candidates.
- Unsupported claims and proof gaps.

## Stage 3: Media Understanding

Goal: Treat media as story material.

Primary skills:

- `skills/media-analysis`
- `skills/motion-direction`

Outputs:

- Media asset reviews.
- Hero candidates.
- Case study visuals.
- Privacy flags.
- Motion potential.

## Stage 4: Creative Direction

Goal: Define the visual and motion language of the site.

Primary skills:

- `skills/design-direction`
- `skills/motion-direction`

Outputs:

- Theme options.
- Selected design system.
- Motion principles.
- Accessibility notes.

## Stage 5: Site Planning and Copy

Goal: Create the actual website blueprint.

Primary skills:

- `skills/site-architecture`
- `skills/copywriting`
- `skills/claim-checking`

Outputs:

- Sitemap.
- Homepage sections.
- Case study plans.
- CTA model.
- Public copy draft.
- Content gaps.

## Stage 6: Build

Goal: Generate a standalone website solution.

Primary skills:

- `skills/code-generation`
- future `skills/github-actions`
- future `skills/firebase-deployment`

Outputs:

- Next.js website source.
- Structured content files.
- Generated artifacts.
- Tests.
- GitHub Actions workflows.
- Firebase Hosting config.
- Deployment docs.

## Stage 7: QA and Release

Goal: Verify the site before publishing.

Primary skills:

- `skills/qa-review`
- `skills/claim-checking`
- future `skills/release-qa`

Outputs:

- QA report.
- Unsupported claim report.
- Privacy report.
- Accessibility notes.
- Performance notes.
- Publish readiness decision.

## Feedback Loop

User feedback should be routed to the smallest affected part of the system.

- Copy feedback goes to copywriting.
- Visual feedback goes to design direction.
- Motion feedback goes to motion direction.
- Structure feedback goes to site architecture.
- Factual feedback goes to claim checking and story extraction.
- Implementation feedback goes to code generation and QA.
